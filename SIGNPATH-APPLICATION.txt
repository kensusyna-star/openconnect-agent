PROJECT DESCRIPTION
──────────────────────────────────────────────────────────────────────────

OpenConnect Agent is an open source Windows agent for remote monitoring
and management (RMM). It provides secure, persistent connections between
managed endpoints and the OpenConnect server platform. The agent enables
IT administrators to remotely manage, monitor, and support Windows
workstations and servers.

Key features:
- Silent installation with no user interaction
- Persistent Windows service for reliable connections
- WebSocket-based real-time communication
- Screen sharing and remote control capabilities
- Secure session management with lifetime session IDs
- Low resource usage (<50MB RAM)
- TLS 1.3 encryption for all communications

Target users are IT administrators, managed service providers (MSPs),
and system administrators who need reliable remote management tools.

WHY DO YOU NEED CODE SIGNING?
──────────────────────────────────────────────────────────────────────────

We need code signing to eliminate Windows SmartScreen warnings that
prevent enterprise deployments. Our target users are IT administrators
and MSPs who require signed executables for security compliance and
organizational policies.

Without code signing:
- Windows SmartScreen blocks downloads with scary warnings
- Chrome/Edge flag downloads as "uncommon" or "dangerous"
- Enterprise security policies prevent installation
- IT departments cannot deploy to managed workstations

Code signing will:
- Enable wider adoption of our open source RMM solution
- Build trust with enterprise customers
- Meet security compliance requirements
- Eliminate barriers to deployment
